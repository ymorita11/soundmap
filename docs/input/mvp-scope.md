# MVP範囲

## MVP定義

### 目的
- 「音だけで旅行気分やリラックスを感じられるか」を検証する
- ユーザーが自発的に環境音を録音・投稿するモチベーションがあるかを確認する
- 寝る前の利用が習慣化するかを観察する

### 対象ユーザー
- 旅行好きで音楽好きな20〜30代の社会人
- SNS疲れを感じている人
- まずは身近な友人・知人から

### 成功の定義
- 初期ユーザーの50%以上が「また使いたい」と回答
- 7日リテンション率15%以上
- 1ユーザーあたり平均3回以上の音声再生 / セッション

## スコープ

### MVP に含める ✅

| 機能 | 理由 |
|------|------|
| 世界地図表示 + ピン表示 | コア体験の土台。地図から音を探す体験がこのアプリの核 |
| 音声再生（30秒・自動停止） | コア体験そのもの。余白を届ける最重要機能 |
| 投稿情報表示（場所名・投稿者名・日時） | 音を聴く前の文脈情報。没入感を高める |
| 音声録音（30秒） | コンテンツ供給の仕組み。聴く体験には投稿が不可欠 |
| プレビュー再生 + 録り直し | 投稿品質を担保するため |
| 位置情報自動取得 + 場所名入力 | 地図にピンを立てるための最低限の情報 |
| ユーザー登録・ログイン | 投稿にはアカウントが必要 |

### MVP に含めない ❌

| 機能 | 理由 | 対応時期 |
|------|------|----------|
| いいね（保存）機能 | コア体験の検証には不要 | Phase 2 |
| 保存一覧画面 | 保存機能と同時に実装 | Phase 2 |
| 投稿履歴 | 最低限は不要 | Phase 2 |
| 通報機能 | 初期は運用で対応 | Phase 2 |
| エリアフィルター | ピン数が少ないうちは不要 | Phase 3 |
| 人気スポット表示 | データ蓄積後 | Phase 3 |
| 高音質再生 | マネタイズ機能 | 将来検討 |
| オフライン保存 | マネタイズ機能 | 将来検討 |
| 季節・時間帯タグ | コンテンツ充実後 | 将来検討 |

## タイムライン（目安）

| フェーズ | 期間 | 内容 |
|----------|------|------|
| MVP | 4〜6週間 | 地図・再生・録音投稿・認証の実装 |
| Phase 2 | MVP後 2〜4週間 | いいね・保存一覧・投稿履歴・通報機能 |
| Phase 3 | Phase 2後 | フィルター・人気表示・プレミアム機能 |

## 制約事項
- 初期はコンテンツ（投稿された音）が少ないため、運営側でシード投稿を用意する必要がある
- 地図APIの利用料が発生する
- 音声ストレージのコストがユーザー数に比例して増加する

## リスクと対策

| リスク | 対策 |
|--------|------|
| 地図上のピンがスカスカ（コールドスタート） | 運営側で主要都市・観光地の音をシード投稿する |
| 不適切な音声の投稿 | MVP段階では手動レビュー、Phase 2で通報機能を追加 |
| 30秒が短すぎる / 長すぎる | ユーザーテストで検証し、秒数を調整 |
| 投稿のモチベーション不足 | 投稿数に応じたバッジや、聴いた人の数を表示するなどを検討 |
