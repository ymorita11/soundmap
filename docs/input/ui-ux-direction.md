# UI/UX方針

## 1. 全体の方向性

### デザインテーマ
ダーク・落ち着き。寝る前に目に優しく、没入感のあるUI。

### キーワード
- 静寂
- 余白
- ダーク
- ミニマル
- 没入

### 参考サービス

| サービス名 | 参考にする点 |
|------------|--------------|
| Headspace | ダークUI、リラックスを促す体験設計 |
| Dark Noise | 環境音アプリのUI、シンプルな操作性 |
| Google Earth | 地図の没入感、探索する楽しさ |
| Muji to Sleep | 余計な情報を削ぎ落としたミニマルUI |

## 2. ビジュアル

### カラー方針
- メインカラー：ダークネイビー / チャコール（#1a1a2e 〜 #16213e 系）
- アクセントカラー：淡いブルーまたはアンバー（ピンや再生ボタンに使用）
- 背景：ダーク（#0a0a0f 〜 #121212）
- テキスト：オフホワイト（#e0e0e0）、セカンダリはグレー（#888）

### タイポグラフィ方針
- 見出し：細めのサンセリフ、控えめな存在感
- 本文：読みやすいサンセリフ、小さめのサイズ
- 雰囲気：静かで上品。主張しすぎないタイポグラフィ

### アイコン・イラスト
- ラインアイコン（細め）を使用
- イラストは使わない。写真やグラフィックも極力使わない
- 地図自体がビジュアルの主役

## 3. UX原則

### 重視する体験

1. **30秒の没入感**：音声再生中は余計な情報を極限まで減らし、音に集中できる状態を作る
2. **地図の探索感**：世界地図をスクロールして「どんな音があるんだろう」とワクワクする体験
3. **操作の最小化**：開く → タップ → 聴く、の3ステップで完結

### 避けたいUX

1. 情報過多（通知、バッジ、数字の表示を最小限に）
2. 中毒的な設計（無限スクロール、自動再生の連続は行わない）

## 4. 画面構成

### 主要画面一覧

| 画面名 | 目的 | 主要要素 |
|--------|------|----------|
| ログイン / 登録 | アカウント認証 | メール入力、SNSログインボタン |
| 地図画面（ホーム） | 音を探して聴く | 世界地図、音声スポットのピン |
| 投稿情報シート | 音の詳細確認・再生 | 場所名、投稿者名、日時、再生ボタン |
| 再生中オーバーレイ | 音に集中する | 残り時間の表示（控えめ）、停止ボタン |
| 録音画面 | 音を録音する | 録音ボタン、タイマー |
| 投稿確認画面 | 録音内容の確認・投稿 | プレビュー再生、場所名入力、投稿ボタン |
| 保存一覧 [MVP後] | 保存した音を再生 | 音声リスト |

### 画面遷移図

```
[ログイン/登録] → [地図画面（ホーム）] → ピンタップ → [投稿情報シート]
                                                           ↓ 再生ボタン
                                                    [再生中オーバーレイ]

                  [地図画面（ホーム）] → ＋ボタン → [録音画面]
                                                       ↓ 録音完了
                                                 [投稿確認画面]
                                                       ↓ 投稿
                                                 [地図画面に戻る]
```

## 5. インタラクション

### アニメーション方針
控えめ。画面遷移やシートの表示にふわっとしたフェードを使う程度。派手な演出は避ける。

### フィードバック
- 再生開始：軽いハプティクス（触覚フィードバック）
- 投稿完了：控えめなサクセスアニメーション
- ピンタップ：ボトムシートがスライドアップ

### エラー表示
- インラインでの控えめなエラー表示
- トーストは使わず、文脈に合った場所にメッセージを表示
- 位置情報取得失敗時は手動で場所を選べるフォールバック
